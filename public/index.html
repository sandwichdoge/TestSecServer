How it works: GET tests attempt to download malicious payloads — your security stack should block the response. POST tests receive an encoded payload from the server, decode it in-browser, then attempt to upload the cleartext sensitive data — your DLP should block the outbound exfiltration. No actual malware is used; all payloads are inert test patterns.
